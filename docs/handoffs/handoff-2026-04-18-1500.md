## Descobertas importantes (drifts expostos pela regeneração do dump)

1. **Migration `2026_04_17_120000_add_document_hash_for_encrypted_search` falhava em MySQL**
   - Tentava `ALTER TABLE customers MODIFY document TEXT` sem dropar antes o índice legado `(tenant_id, document)`. Criada **migration 500001** de reparo (idempotente).

2. **Laravel `->change()` via doctrine/dbal perde DEFAULTs em MySQL**
   - Migration `2026_04_17_220000_normalize_monetary_precision` deixou `accounts_payable.amount_paid` e `accounts_receivable.amount_paid` sem default em MySQL. SQLite mascarou porque `->change()` é no-op. Criada **migration 500002** restaurando default `'0.00'`.

3. **Regex `_\w+'` no `generate_sqlite_schema.php` era agressivo demais**
   - Comia `_Paulo'` de `'America/Sao_Paulo'`. Restringido a whitelist de charsets conhecidos (`utf8|utf8mb4|latin1|...`).

4. **`InmetroReferenceSeeder` não passava `tenant_id` no upsert de `inmetro_locations`**
   - Schema real em MySQL é `NOT NULL`; dump antigo mascarava como NULLABLE.

5. **`StoreWorkOrderRequest.rules.priority` desalinhado com enum canônico**
   - Commit `a050fae` (Onda 8 parte A) migrou `normal→medium` em vários lugares mas esqueceu do FormRequest. 9 testes também estavam enviando `'normal'` — todos corrigidos.
