## Riscos conhecidos

1. **Alterar `AuditLog.$fillable` quebra callers** — qualquer código que faça `AuditLog::create([...tenant_id..., ...user_id...])` em vez de `AuditLog::log()` vai falhar. Grep amplo obrigatório antes.

2. **sec-26 email_verified** pode quebrar login de usuários legados (`email_verified_at = NULL` por não terem confirmado historicamente). Mitigação: migration que seta `email_verified_at = created_at` para users criados antes de 2026-04-19, OU flag `FORCE_EMAIL_VERIFICATION=false` em produção.

3. **sec-08 cross-check user↔token binding** implica persistir `issued_for_tenant_id` no PersonalAccessToken — mudança de schema grande.

4. **sec-01 CSP nonce** requer frontend React cooperar (inline scripts precisam nonce). Fix backend sozinho quebra SPA.

5. **MySQL local desatualizado** — 20+ migrations pendentes desde 2026-04-17. Uma delas falha com "cannot add a STORED column". Resolver antes de regenerar schema dump.
