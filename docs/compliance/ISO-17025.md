# Conformidade Legal - Norma ISO-17025 (Metrologia)

> **[AI_RULE]** O módulo Lab requer total clareza da Cadeia de Custódia e Incerteza Matemática. Dados soltos perdem o selo ABNT/INMETRO.
> **[COMPLIANCE]** Ver documentação técnica do módulo: [Lab](../modules/Lab.md)

## 1. Assinatura e Custódia Documental `[AI_RULE_CRITICAL]`

> **[AI_RULE_CRITICAL] Princípio do Double Sign-Off Condicional**
> **Atenção:** Esta regra é configurável! A IA DEVE envolver essa validação em um if (`TenantSetting::isFeatureEnabled('strict_iso_17025')`).
>
> *Se Habilitado:* Laudos de calibração gerados que baseiam os equipamentos dos clientes não possuem validade sem assinatura revisacional. A transição para "APPROVED" exige um `supervisor_user_id` diferente do Técnico criador.
> *Se Desabilitado:* O Controller aceita a mudança para "APPROVED" com apenas 1 assinatura do técnico padrão, simplificando o processo.

### 1.1 Exemplo de Código — Dual Signature `[AI_RULE_CRITICAL]`

```php
// CalibrationCertificateService::approve()
public function approve(CalibrationCertificate $cert, int $supervisorId): void
{
    if (TenantSetting::isFeatureEnabled('strict_iso_17025')) {
        if ($cert->technician_user_id === $supervisorId) {
            throw new BusinessException(
                'ISO-17025 §7.2.1: O revisor deve ser diferente do técnico executor.'
            );
        }
        if (!UserCompetency::hasValid($supervisorId, $cert->instrument->type)) {
            throw new BusinessException(
                'ISO-17025 §6.2: Revisor sem competência válida para este tipo de instrumento.'
            );
        }
    }

    $cert->update([
        'supervisor_user_id' => $supervisorId,
        'status' => 'approved',
        'approved_at' => now(),
    ]);

    AuditLog::record('approve', $cert, ['supervisor' => $supervisorId]);
}
```

## 2. Rastreabilidade Ambiental (Condições Físicas)

Uma Medição de Calibração perde todo o valor e o Inmetro anula a emissão (`InmetroSeal`) se os inputs do Ambiente não tiverem sido declarados. Qualquer Endpoint de inserção metrológica DEVE validar obrigatoriamente e cruzar com um hardware de Temperatura, Pressão e Umidade instanciado em `LabLogbookEntry`.
O banco não pode aceitar inserts órfãos.

### 2.1 Variáveis Ambientais Obrigatórias

| Variável | Unidade | Faixa Típica | Model |
|---|---|---|---|
| Temperatura | °C | 20 ± 2 | `LabLogbookEntry` |
| Umidade Relativa | %RH | 45–65 | `LabLogbookEntry` |
| Pressão Atmosférica | hPa | 1013 ± 30 | `LabLogbookEntry` |

- Registros ambientais devem ser vinculados ao `CalibrationCertificate` no momento da medição.
- Se a faixa estiver fora da tolerância configurada pelo tenant (`TenantSetting::get('lab_env_tolerance')`), o sistema emite alerta e bloqueia a aprovação do certificado.

### 2.2 Exemplo de Código — Validação Ambiental `[AI_RULE_CRITICAL]`

```php
// LabLogbookEntry — validação antes de vincular ao certificado
public function validateEnvironmentalConditions(): void
{
    $tolerance = TenantSetting::get('lab_env_tolerance', [
        'temperature' => ['min' => 18, 'max' => 22],
        'humidity'    => ['min' => 45, 'max' => 65],
        'pressure'    => ['min' => 983, 'max' => 1043],
    ]);

    foreach (['temperature', 'humidity', 'pressure'] as $var) {
        $value = $this->{$var};
        if ($value < $tolerance[$var]['min'] || $value > $tolerance[$var]['max']) {
            throw new EnvironmentalOutOfRangeException(
                "ISO-17025 §7.7.1: {$var} = {$value} fora da faixa permitida."
            );
        }
    }
}
```

## 3. Certificado de Calibração (`CalibrationCertificate`)

O certificado é o produto final do laboratório e deve conter obrigatoriamente:

| Campo | Obrigatoriedade | Descrição |
|---|---|---|
| `certificate_number` | Obrigatório | Numeração sequencial única por tenant |
| `instrument_id` | Obrigatório | Referência ao instrumento calibrado (`Instrument`) |
| `calibration_date` | Obrigatório | Data da execução da calibração |
| `next_calibration_date` | Obrigatório | Data prevista para recalibração |
| `measurement_results` | Obrigatório | JSON com leituras, padrões e desvios |
| `expanded_uncertainty` | Obrigatório | Incerteza expandida (U) com fator de abrangência k |
| `coverage_factor` | Obrigatório | Fator k (tipicamente k=2 para 95,45% de confiança) |
| `environmental_conditions` | Obrigatório | Snapshot das condições ambientais via `LabLogbookEntry` |
| `technician_user_id` | Obrigatório | Técnico executor |
| `supervisor_user_id` | Condicional | Revisor (obrigatório se `strict_iso_17025` habilitado) |
| `status` | Obrigatório | `draft → reviewed → approved → issued → cancelled` |

### 3.1 Incerteza de Medição

- Toda calibração DEVE declarar a incerteza expandida (U) conforme GUM (Guide to the Expression of Uncertainty in Measurement).
- O cálculo considera: repetibilidade, resolução do instrumento, incerteza do padrão, contribuições ambientais.
- O campo `measurement_results` armazena o balanço de incertezas completo em JSON estruturado.
- O fator de abrangência k deve ser explícito no certificado emitido.

## 4. Rastreabilidade de Equipamentos e Padrões

### 4.1 Cadeia de Rastreabilidade Metrológica

- Todo `Instrument` usado como padrão de referência deve possuir certificado de calibração válido emitido por laboratório acreditado (RBC/Cgcre ou equivalente internacional).
- O sistema valida automaticamente: se o padrão de referência está com calibração vencida, a calibração do instrumento do cliente é bloqueada.
- Campos obrigatórios no `Instrument`: `serial_number`, `manufacturer`, `model`, `calibration_due_date`, `traceability_certificate`.

### 4.2 Controle de Validade

- Cron job diário verifica `Instrument.calibration_due_date` e envia notificações 30, 15 e 7 dias antes do vencimento.
- Instrumentos vencidos recebem flag `is_expired = true` e são impedidos de serem usados em novas calibrações.

## 5. Controle de Documentos e Registros

- Certificados emitidos (`status = issued`) são **imutáveis** — qualquer correção gera uma nova revisão (`CalibrationCertificateRevision`).
- O sistema mantém histórico completo de revisões com `revision_number`, `reason`, `changed_by`, `changed_at`.
- PDFs gerados são armazenados no storage com hash SHA-256 para garantia de integridade.
- Tempo de retenção mínimo configurável por tenant (padrão: 5 anos conforme ABNT NBR ISO/IEC 17025:2017 §8.4).

## 6. Trilha de Auditoria (Audit Trail)

- Toda ação no módulo Lab é registrada em `audit_logs`: criação, edição, aprovação, emissão, cancelamento.
- Campos auditados: `user_id`, `action`, `model_type`, `model_id`, `old_values`, `new_values`, `ip_address`, `timestamp`.
- Registros de auditoria são **imutáveis** e **não deletáveis** — apenas consulta é permitida.
- Auditorias internas e externas podem ser rastreadas via filtros por período, usuário e tipo de ação.

## 7. Competência Técnica

- Técnicos que executam calibrações devem ter registro de competência no sistema (`UserCompetency`).
- Campos: `user_id`, `skill_type` (ex: calibração dimensional, elétrica, térmica), `certification_date`, `expiry_date`, `evidence_file`.
- O sistema bloqueia atribuição de calibração a técnico sem competência válida para o tipo de instrumento, quando `strict_iso_17025` está habilitado.
- Registros de treinamento e qualificação ficam vinculados ao perfil do usuário.

### 10.4 Fórmula GUM de Incerteza (§7.5) `[AI_RULE_CRITICAL]`

> **[AI_RULE_CRITICAL]** Todas as operações de cálculo de incerteza DEVEM usar `bcmath` para precisão decimal. O uso de operações float nativas do PHP é PROIBIDO neste contexto.

| Camada | Artefato | Detalhes |
|---|---|---|
| Service | `UncertaintyCalculationService` | Implementa o cálculo GUM (Guide to the Expression of Uncertainty in Measurement) |
| Fórmula | Incerteza combinada | `u_c = sqrt(sum(c_i² × u_i²))` onde `c_i` são os coeficientes de sensibilidade e `u_i` são as incertezas padrão de cada fonte |
| Fórmula | Incerteza expandida | `U = k × u_c` onde `k` é o fator de abrangência (tipicamente k=2 para 95,45% de confiança) |
| Fields | `CalibrationCertificate` | `measurement_results` (JSON com balanço de incertezas: fontes, valores, distribuições, graus de liberdade), `expanded_uncertainty` (decimal via bcmath), `coverage_factor` (decimal, padrão 2.0), `confidence_level` (decimal, padrão 95.45) |
| Precisão | `bcmath` obrigatório | Todas as operações aritméticas usam `bcadd()`, `bcsub()`, `bcmul()`, `bcdiv()` e implementação customizada `bcSqrt()` (Newton-Raphson) com scale mínimo de 10 casas decimais |
| Teste | `UncertaintyCalculationTest` | Valida: cálculo com fontes conhecidas vs. resultado esperado, precisão decimal, fator de abrangência, nível de confiança |

### 10.5 Implementação de bcSqrt (raiz quadrada com bcmath) `[AI_RULE_CRITICAL]`

> **Atenção:** `bcsqrt()` NÃO existe no PHP bcmath nativo. Usar a implementação abaixo via `UncertaintyCalculationService`.

```php
// Service: App\Services\Lab\UncertaintyCalculationService

/**
 * Calcula raiz quadrada com precisão arbitrária usando método de Newton-Raphson
 * @param string $operand Número como string para bcmath
 * @param int $scale Casas decimais de precisão (padrão: 20)
 */
public function bcSqrt(string $operand, int $scale = 20): string
{
    if (bccomp($operand, '0', $scale) === 0) return '0';
    $x = bcdiv($operand, '2', $scale);
    for ($i = 0; $i < 100; $i++) {
        $nx = bcdiv(bcadd($x, bcdiv($operand, $x, $scale), $scale), '2', $scale);
        if (bccomp($x, $nx, $scale) === 0) break;
        $x = $nx;
    }
    return $x;
}
```

### 11.5 Exemplo de Soft Delete — Imutabilidade `[AI_RULE_CRITICAL]`

```php
// CalibrationCertificate — NUNCA deletar, apenas cancelar com motivo
class CalibrationCertificate extends Model
{
    use SoftDeletes, BelongsToTenant;

    // Certificados emitidos não podem ser soft-deleted
    protected static function booted(): void
    {
        static::deleting(function (self $cert) {
            if ($cert->status === 'issued') {
                throw new BusinessException(
                    'ISO-17025 §8.4: Certificados emitidos são imutáveis. Use cancelamento com revisão.'
                );
            }
        });
    }

    public function cancel(string $reason, int $cancelledBy): CalibrationCertificateRevision
    {
        $revision = $this->revisions()->create([
            'revision_number' => $this->revisions()->count() + 1,
            'reason' => $reason,
            'changed_by' => $cancelledBy,
            'changed_at' => now(),
            'previous_status' => $this->status,
        ]);

        $this->update(['status' => 'cancelled']);
        AuditLog::record('cancel', $this, ['reason' => $reason]);

        return $revision;
    }
}
```
